## Philosophie du cours

La philosophie du cours est basée sur les principes suivants :

- **Straight to the point**: ne pas perdre de temps, être rapide, efficace et
  utile.
- **Open**: les contenus peuvent être améliorés, modifiés et discutés
  ouvertement.
- **Resilient**: les matériaux doivent être compréhensibles et maintenables par
  d'autres personnes.
- **Freedom**: les personnes qui étudient doivent pouvoir poursuivre les sujets
  qui les intéressent et les engagent.
- **Shit happens**: créer un espace sûr où les erreurs sont acceptables et où
  les personnes qui étudient peuvent discuter ouvertement des défis.

### Principes pédagogiques

Les éléments pédagogiques suivants ont été utiles pour façonner ce cours :

- _"[La compétence est] un savoir-agir complexe prenant appui sur la
  mobilisation et la combinaison efficaces d'une variété de ressources internes
  et externes à l'intérieur d'une famille de situations."_ - Jacques Tardif
- _"L'intelligence ce n'est pas ce que l'on sait mais ce que l'on fait quand on
  ne sait pas."_ - Jean Piaget
- [Bloom's taxonomy](https://en.wikipedia.org/wiki/Bloom%27s_taxonomy)

### Approche d'apprentissage

- **Self-directed learning**: donner aux personnes qui étudient les outils pour
  apprendre de manière indépendante.
- **Big picture first**: de courtes présentations donnent un aperçu, puis les
  personnes qui étudient explorent plus en profondeur à travers des matériaux
  pratiques.
- **Flexible pace**: les personnes qui étudient peuvent choisir leur propre
  parcours : théorie d'abord, pratique d'abord ou mixte.
- **Hybrid-friendly**: tous les matériaux disponibles pour l'apprentissage sur
  site ou à distance.
- **Real-world relevance**: se concentrer sur les compétences pratiques
  utilisées dans l'industrie, pas seulement des exercices académiques.
- _"Teach them to fish"_: guider les personnes qui étudient à trouver des
  solutions plutôt que de donner des réponses directes.

### Fondements pédagogiques

L'idée derrière les courtes présentations + matériel de cours est la suivante :
présenter rapidement la vue d'ensemble à travers des diapositives qui illustrent
le sujet. Cela donne un aperçu, la grande image que **les personnes qui étudient
doivent approfondir par elles-mêmes**. Ensuite, les personnes qui étudient
passent à un matériel de cours où elles peuvent se salir les mains. Les
personnes peuvent approfondir le cours à leur propre rythme et je suis ensuite
disponible pour aider celles qui pourraient avoir d'autres questions/besoin de
plus d'explications.

L'idée principale derrière cela est de permettre aux personnes d'avancer à leur
propre rythme : si quelqu'un veut commencer par la partie théorique du cours
avant de commencer le contenu pratique, il le peut. Si quelqu'un veut "apprendre
en faisant", il le peut. Un mélange entre les deux est également possible. Cela
permet également le travail hybride sur site/hors site : si les personnes ne
peuvent pas venir à mon cours, ce n'est pas un problème, elles ont tout ce dont
elles ont besoin pour apprendre par elles-mêmes, à la maison, à un autre moment,
etc.

L'objectif du cours est d'apprendre quelque chose d'utile (pour soi, pour ses
(futurs) emplois, pour travailler avec les autres, etc.). Il doit refléter ce
qui est utilisé dans la vraie vie, pas quelque chose qui est oublié après la fin
du cours.

Au final, c'est un effort d'équipe : je fournis le matériel, les personnes qui
étudient apprennent, donnent des retours et je l'améliore. Je suis là pour les
aider si elles en ont besoin. Mais je ne leur donne pas le poisson, je leur
apprends à pêcher.

## Avant le cours

- Examiner la revue du semestre, les évaluations GAPS et les retours des
  semestres précédents ainsi que les
  [Issues](https://github.com/heig-vd-progim2-course/heig-vd-progim2-course/issues).
- Mettre à jour le matériel de cours et les présentations à partir des retours.
- Revoir les diapositives et la séquence du cours.
- Envoyer un mail à toutes les personnes qui étudient pour leur rappeler le
  cours avec :
  - Un lien vers le dépôt GitHub.
  - Quand et où le cours aura lieu.
- Ajouter les évaluations et mini-projets à [GAPS](https://gaps.heig-vd.ch/).
- Imprimer la feuille de présence sur [GAPS](https://gaps.heig-vd.ch/).
  Sélectionner le format imprimable et imprimer le PDF sans l'en-tête et le pied
  de page.
- Créer les formulaires de retours sur [Framaform](https://framaforms.org/) pour
  les retours pendant le cours (avant les évaluations GAPS).
- Configurer Git pour le développement en équipe dans le script
  `.devcontainer/post-create.sh`.

## Configuration de la manette 8bitdo Micro

Pour les besoins du cours, nous allons utiliser une manette de jeu 8bitdo. Voici
la configuration recommandée pour les boutons de la manette, qui correspond à
l'utilisation dans le cadre du cours. Les boutons sont configurés pour faciliter
la navigation et les interactions pendant les sessions de cours.

Il s'agit de cette manette : [8bitdo Micro](https://www.8bitdo.com/micro/)

### Comment utiliser le fichier de configuration

Le fichier `./configuration-manette.gamecontroller.amgp` est un profil à charger
dans l'application AntiMicroX.

Pour reproduire la configuration :

- Branchez ou associez la manette 8bitdo Micro à l'ordinateur.
- Ouvrez AntiMicroX.
- Sélectionnez la manette détectée dans l'application.
- Chargez le profil `./configuration-manette.gamecontroller.amgp` depuis le menu
  d'ouverture de profil de l'application.
- Vérifiez que les boutons correspondent bien au schéma ci-dessous, puis
  enregistrez si vous devez adapter localement la configuration.

```
 L1 L2            R2 R1
   U      -  +      X
 L   R            Y   A
   D      S  H      B
```

**Légende**

- Côté gauche.
  - L1 : <kbd>Ctrl</kbd> + <kbd>Alt</kbd> + <kbd>Plus (+)</kbd>.
  - L2 : <kbd>Ctrl</kbd> + <kbd>Alt</kbd> + <kbd>Minus (-)</kbd>.
  - U : <kbd>Left</kbd>.
  - L : <kbd>Page Up</kbd>.
  - R : <kbd>Page Down</kbd>.
  - D : <kbd>Right</kbd>.
- Milieu.
  - \- : <kbd>Ctrl</kbd> + <kbd>Shift</kbd> + <kbd>Tab</kbd>.
  - \+ : <kbd>Ctrl</kbd> + <kbd>Tab</kbd>.
  - S : Aucun.
  - H : <kbd>F</kbd>.
- Côté droit.
  - R2 : <kbd>Ctrl</kbd> + <kbd>Minus (-)</kbd>.
  - R1 : <kbd>Ctrl</kbd> + <kbd>Plus (+)</kbd>.
  - X : <kbd>Left</kbd>.
  - Y : <kbd>Down</kbd>.
  - A : <kbd>Up</kbd>.
  - B : <kbd>Right</kbd>.
