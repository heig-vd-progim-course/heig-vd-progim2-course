## Objectif pédagogique

Ce dossier contient des exemples de code **incorrects** montrant les erreurs les
plus fréquentes commises par les personnes qui débutent en programmation.

Chaque fichier :

- Commence par le préfixe `ERROR-` pour le distinguer des exemples corrects.
- Contient du code qui **ne fonctionne pas** ou **produit un comportement
  incorrect**.
- Inclut des commentaires détaillés expliquant :
  - Quelle est l'erreur.
  - Pourquoi elle se produit.
  - Quel message d'erreur ou comportement inattendu cela génère.
  - Comment la corriger (explication).
  - Où trouver des exemples corrects dans le dossier `01-exemples/`.

## Comment utiliser ces exemples

1. **Lisez les commentaires** dans chaque fichier avant d'exécuter le code.
2. **Essayez de prédire** quel sera le problème.
3. **Exécutez le code** pour observer l'erreur.
4. **Comprenez** pourquoi l'erreur se produit grâce aux commentaires.
5. **Consultez** les exemples corrects dans `01-exemples/` pour voir la bonne
   façon de faire.

> [!IMPORTANT]
>
> L'objectif n'est **pas** de mémoriser les erreurs, mais de développer votre
> capacité à :
>
> - Reconnaître les signes avant-coureurs d'un problème.
> - Lire et comprendre les messages d'erreur.
> - Raisonner pour trouver la cause du problème.
> - Appliquer les bonnes pratiques pour éviter ces erreurs.
